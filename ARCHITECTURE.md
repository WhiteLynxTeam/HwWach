# Архитектура приложения HwWach

## Обзор

HwWach - это веб-приложение на Go, предназначенное для управления устройствами и фотографиями. Приложение построено по принципам многослойной архитектуры с четким разделением ответственности между слоями.

## Описание слоев

1. **External Layer** - внешние зависимости: клиенты HTTP, база данных PostgreSQL и хранилище файлов MinIO
2. **Presentation Layer** - слой представления: маршрутизатор Gin, промежуточное ПО (JWT) и конфигурация
3. **Interface Layer** - интерфейсный слой: обработчики HTTP-запросов (DeviceHandler, PhotoHandler, RequestHandler)
4. **Business Layer** - бизнес-логика: сервисы, реализующие бизнес-правила (DeviceService, PhotoService, RequestService)
5. **Data Access Layer** - доступ к данным: репозитории, обеспечивающие доступ к базе данных
6. **Domain Layer** - доменный слой: модели данных, представляющие предметную область

## Аутентификация

Приложение не реализует собственную логику регистрации и авторизации пользователей. 
Аутентификация выполняется внешним сервисом, который выдаёт JWT-токены. 
Приложение проверяет входящий JWT-токен через middleware и извлекает `user_id` из claims токена.

## Технологии

- **Gin-Gonic** - веб-фреймворк для маршрутизации HTTP-запросов
- **GORM** - ORM для работы с базой данных PostgreSQL
- **MinIO** - распределенное хранилище объектов для хранения фотографий
- **Viper** - управление конфигурацией приложения
- **JWT** - проверка токенов от внешнего сервиса аутентификации

## Состояние проекта

На момент анализа проект находится на начальной стадии разработки - многие обработчики содержат заглушки и требуют реализации.
